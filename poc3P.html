POC 3: The 24 game

Deze game heb ik als kind van mijn ouders geleerd. Dit is echt een typishe chinese kaarten spel. Ik heb gekozen om deze game te maken als poc 3, omdat dit vrijwel een mogelijk game is. Hiermee wil ik mijzelf pushen om na te denken en beter te worden met javascript programmeren. De source code kan je kijken op Fontys Gitlab onder naam van 'poc3'. Je kan hier ook de Demo bekijken.

Spel regel

De regels is:

  • Er is 4 spelers.
  • Er wordt gewerkt met 4 keer 10 kaarten van 1 tot 10.
  • Een ieder speler krijg tien willekeurige kaarten.
  • Een ieder speler geeft elke keer 1 kaart uit, dus in totaal 4 kaarten op tafel.
  • Probleem: hoe kom je d.m.v optelen, aftrekken, vermenigvuldigen of delen met deze 4 kaarten op 24. Elk kaart mag alleen 1 keer gebruikt worden.
    Voorbeeld: de 4 kaarten zijn 6, 1, 3 en 4.
    Dan is de oplossing: 6/ (1-3/4) = 24
  • Wie op de eerste oplossing komt is de winner van die ronde.
  • Dit game eindigt wanneer niemand nog kaarten over heeft.
  • De uiteindelijke winnaar is degene die de meeste rondes gewonnen heeft.

Aanpak

Om te beginnen met bouwen van dit game heb ik eerst de spel regels geschreven. Daarna heb ik stap voor stap alle details geschreven in de comments over wat ik wil maken, hoe ik het gaat maken en wat ik nodig heb. Zo heb ik eerst alle plaatjes opgezocht en opgeslagen in een Array. Vervolgens heb ik de layout van het game opgezet. Daarna ben ik aan de slag gegaan met eerste doel in de comment.

Na de eerste doel ben ik naar de tweede doel gegaan. Hier zat ik namelijk 2-3 dagen vast. Ik heb mijn logica als comment opgeschreven bij regel 214 tot 216. Dus ik snap de logica wel maar de vertaling van tekst naar code gaat mis. Ik was ook geweest naar Frank voor feedback. Hij vindt dat deze een hele goeie proof of concept is. Hij heeft samen met mij naar de logica gekeken wat ik wil bereiken. De logica is goed, alleen om op de code komen heb ik wat tijd nodig.
Na heel veel onderzoek naar hoe ik een random element uit een array weg kan halen en die kan opslaan in een andere array en voorkomen dat niemand dezelfde kaarten krijgt, ben ik opnieuw naar mijn logica gegaan. Toen realiseer ik een detail. Wanneer je in realiteit kaarten speelt, shuffle je pak kaarten eerst. Dus daar is mijn denk fout geweest. Ik moest eerst de array met de hele pak kaarten willekeurig maken en daarna die verdelen aan 4 (array's) speler zodoende heeft een ieder willekeurige en niet overeenkomde kaarten.

De derde doel is om bij elke druk op add my card button één van de omgedraaide kaarten weg te halen en deze in het midden van de scherm te plaatsen. Hier heb ik met de kennis van poc 2 de buttons laten deactiveren en activeren wanneer het moet zodat er niet meerdere keren op een button gedrukt kan worden. Daarnaast heb ik een onderzoek gedaan naar Parent-Child-Siblings relationship om de verband in HTML te begrijpen en hoe je dit kan veranderen met Javascript. Met deze kennis heb ik het probleem opgelost van het weghalen van één omgedraaide kaart.

Feedback

Frank: "Hele goeie proof of concept! Voor mij hoeft de hele kaarten spel niet af, maar als je de array probleem kan oplossen is het goed genoeg."

Wat heb ik geleerd?

Ik heb geleerd:

  • hoe ik een for loop kan gebruiken.
  • hoe ik array kan randomizen.
  • hoe elementen van de ene Array in een ander Array kan opslaan.
  • Als je vast zit met vertalen van logica naar code, ga eens helemaal opnieuw beginnen om te kijken of de fout weleens bij de logica zit.
  • wat Parent-Child-Siblings relationship is en hoe je dat kan veranderen met JavaScript.
  • hoe ik een child element weg kan halen
  • hoe ik plaatjes in een Array in volgorde kan laten zien.
